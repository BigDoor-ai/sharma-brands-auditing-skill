# Website Conversion Audit Checklist

Use this checklist whenever you audit or improve a website.

## 1. 3-Second Clarity

- [ ] Can a cold visitor understand what is being sold within 3 seconds?
- [ ] Is the target buyer clear?
- [ ] Is the outcome clear?
- [ ] Is the offer specific?
- [ ] Is the next step obvious?
- [ ] Is there a clear visual explaining the product/service?

## 2. Hero Section

- [ ] Clear headline
- [ ] Outcome-focused subheadline
- [ ] Product/service/workflow visual
- [ ] Primary CTA
- [ ] Trust signal near CTA
- [ ] Optional anxiety-reducing secondary CTA
- [ ] No vague slogan without explanation

## 3. CTA Quality

- [ ] CTA is specific
- [ ] CTA describes the next action
- [ ] CTA appears above the fold
- [ ] CTA repeats after major sections
- [ ] CTA is mobile-visible
- [ ] CTA microcopy reduces anxiety where useful

## 4. Trust and Proof

- [ ] Proof appears early
- [ ] Proof appears throughout the page
- [ ] Client logos are real or marked as placeholders
- [ ] Testimonials are real or marked as placeholders
- [ ] Metrics are verified or omitted
- [ ] Case studies are linked or clearly summarized
- [ ] Proof answers buyer objections

## 5. Buyer Education

- [ ] Page explains what the product/service is
- [ ] Page explains who it is for
- [ ] Page explains how it works
- [ ] Page explains what happens after the CTA
- [ ] Page explains delivery/setup/support
- [ ] FAQs answer real objections

## 6. Friction Removal

- [ ] No unnecessary navigation away from campaign goal
- [ ] No dead clicks
- [ ] No confusing button hierarchy
- [ ] No hidden pricing direction when pricing matters
- [ ] No long forms without reason
- [ ] No important information buried on other pages
- [ ] No vague next step

## 7. Mobile-First UX

- [ ] Hero readable on mobile
- [ ] CTA visible early
- [ ] Tap targets are large
- [ ] Paragraphs are short
- [ ] No horizontal scroll
- [ ] Forms are easy to complete
- [ ] Visuals are optimized
- [ ] Sticky CTA considered

## 8. Forms and Lead Capture

- [ ] Only asks for required information
- [ ] Explains what happens after submission
- [ ] Has clear error states
- [ ] Works on mobile
- [ ] Avoids asking for high-friction data too early

## 9. Pricing and Offer

- [ ] Pricing is visible when possible
- [ ] If exact pricing is unknown, pricing factors are explained
- [ ] Packages or offer details are clear
- [ ] What is included is clear
- [ ] What is excluded is clear
- [ ] Urgency is real, if used

## 10. Technical Basics

- [ ] SEO title exists
- [ ] Meta description exists
- [ ] Open Graph metadata exists
- [ ] Page loads fast enough for target audience
- [ ] Images have useful alt text
- [ ] Semantic HTML is used
- [ ] Analytics placeholders exist where relevant
